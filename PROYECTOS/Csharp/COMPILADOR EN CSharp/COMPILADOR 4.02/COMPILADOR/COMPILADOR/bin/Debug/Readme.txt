OBJETIVO.

Reconocer las fases que conforman un compilador y su implementación.

ELEMENTOS DEL COMPILADOR.
Este compilador cuenta con 4 fases visuales y 5 fases principales. Las primeras 3 fases se basan en Análisis del código, el cual se conforma de 3 fases:

ANALISIS LEXICO:
Encargado de encontrar problemas en las palabras ingresadas denominados TOKENS.

ANALISIS SINTACTICO:
Encargado de encontrar problemas en el orden que tienen las declaraciones.

ANALISIS SEMANTICO:
Encargado de encontrar problemas en el sentido de la sintaxis.

Posteriormente a estas fases se encuentran las dos siguientes:

GENERACION DE CODIGO INTERMEDIO (POLACA):

Código que nos permite implementar código ensamblador, puesto que su estructura nos permite leer como el código esta conformado y sus operaciones

GENERACION DE CODIGO ENSAMBLADOR:

Código final que nos permite ejecutar en código ensamblador (en este caso ocupamos EMU8086).


OBJETIVO PRINCIPAL:

El objetivo principal de este compilador no es como tal el crear un compilador funcional para la programación, si no una demostración del funcionamiento
de un compilador.

FUNCIONAMIENTO.

Para poder utilizar el compilador en CSharp debemos tener la siguiente sintaxis:

Program (CUALQUIER NOMBRE QUE SE DESEE) ;

Var
int (VARIABLES DE TIPO ENTERO QUE DESEE, SEPARADAS POR COMAS) ;
float (VARIABLES DE TIPO FLOTANTE QUE DESEE, SEPARADAS POR COMAS) ;

Begin
(INICIALIZACION DE TODAS LAS VARIABLES DECLARADAS CON SU VALOR DESEADO) ;

(SENTENCIA QUE DESEA IMPLEMENTAR, EN ESTE COMPILADOR SOLO SE PUEDE HACER UNA SOLA SENTENCIA DE VARIAS OPERACIONES ARITMETICAS) ;

End ;
